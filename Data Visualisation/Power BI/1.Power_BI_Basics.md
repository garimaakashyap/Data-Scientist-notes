# Power BI Basics - Complete Guide

## Introduction
Power BI is Microsoft's business intelligence tool for data visualization and analytics. It helps transform data into interactive, visual insights.

## What is Power BI?
Power BI is a collection of software services, apps, and connectors that work together to turn unrelated data sources into coherent, visually immersive, and interactive insights.

## Power BI Components

1. **Power BI Desktop** - Free desktop application for creating reports
2. **Power BI Service** - Cloud-based service for sharing and collaboration
3. **Power BI Mobile** - Mobile apps for viewing reports on the go

## Installation

### Download Power BI Desktop
1. Go to https://powerbi.microsoft.com/desktop/
2. Download and install Power BI Desktop (free)
3. Launch the application

## 1. Getting Started

### Power BI Desktop Interface

**Main Components:**
- **Ribbon** - Top menu with tools and options
- **Fields Pane** - Lists all tables and fields from your data
- **Visualizations Pane** - Different chart types you can create
- **Canvas** - Where you build your visualizations
- **Filters Pane** - Apply filters to your report
- **Data View** - View and edit your data tables
- **Model View** - See relationships between tables

### Creating Your First Report

**Step 1: Get Data**
1. Click "Get Data" button
2. Choose your data source (Excel, CSV, Database, etc.)
3. Select and load your data

**Step 2: Create Visualization**
1. Select a chart type from Visualizations pane
2. Drag fields to appropriate areas (Axis, Values, Legend)
3. Customize the visualization

**Step 3: Save Report**
1. Click File → Save
2. Save as .pbix file

### Import vs DirectQuery

**Import Mode:**
- Data is copied into Power BI
- Faster performance
- Works offline
- Limited by file size (1GB compressed)
- Data refresh required

**DirectQuery Mode:**
- Connects directly to source
- Always up-to-date
- No size limit
- Slower performance
- Requires live connection

## 4. Basic DAX (Data Analysis Expressions)

### What is DAX?
DAX is a formula language used in Power BI to create calculated columns, measures, and custom tables.

### Calculated Columns vs Measures

**Calculated Columns:**
- Added to table as new column
- Calculated row by row
- Uses table storage
- Use for filtering, grouping, relationships

**Measures:**
- Calculated on-the-fly
- Context-aware
- Doesn't use storage
- Use for aggregations, calculations

### Basic DAX Functions

**1. SUM**
```dax
Total Sales = SUM(Sales[Amount])
```
- Adds up all values in a column

**2. AVERAGE**
```dax
Average Sales = AVERAGE(Sales[Amount])
```
- Calculates average of values

**3. COUNT**
```dax
Total Orders = COUNT(Sales[OrderID])
```
- Counts number of rows

**4. COUNTROWS**
```dax
Total Rows = COUNTROWS(Sales)
```
- Counts rows in a table

**5. MAX/MIN**
```dax
Max Sales = MAX(Sales[Amount])
Min Sales = MIN(Sales[Amount])
```
- Finds maximum/minimum value

**6. IF Statement**
```dax
Sales Category = IF(Sales[Amount] > 1000, "High", "Low")
```
- Conditional logic

**7. CALCULATE**
```dax
Sales 2023 = CALCULATE(SUM(Sales[Amount]), Sales[Year] = 2023)
```
- Modifies filter context

### Creating Your First Measure

**Steps:**
1. Right-click table in Fields pane
2. Select "New measure"
3. Enter formula:
```dax
Total Revenue = SUM(Sales[Amount])
```
4. Press Enter
5. Use measure in visualizations

## 5. Data Transformation (Power Query)

### What is Power Query?
Power Query is the data transformation engine in Power BI. It helps clean, transform, and shape your data.

### Opening Power Query Editor

**Steps:**
1. Click "Transform Data" button
2. Power Query Editor opens
3. See your data and transformation steps

### Common Transformations

**1. Remove Columns**
- Select columns → Right-click → "Remove Columns"

**2. Rename Columns**
- Double-click column name → Type new name

**3. Change Data Type**
- Click data type icon → Select new type
- Common types: Text, Whole Number, Decimal, Date

**4. Remove Duplicates**
- Select columns → Right-click → "Remove Duplicates"

**5. Filter Rows**
- Click filter icon on column
- Select conditions (equals, contains, greater than, etc.)

**6. Split Columns**
- Select column → Right-click → "Split Column"
- Choose delimiter (space, comma, etc.)

**7. Add Custom Column**
- Add Column tab → Custom Column
- Enter formula:
```
[Column1] & " " & [Column2]
```

**8. Replace Values**
- Select column → Right-click → "Replace Values"
- Enter old and new values

### Applied Steps

**Viewing Steps:**
- Right side shows "Applied Steps"
- Each transformation is a step
- Can delete or modify steps
- Steps are executed in order

## 8. Relationships

### What are Relationships?
Relationships connect tables in your data model, allowing you to analyze data across tables.

### Types of Relationships

**1. One-to-Many (Most Common)**
- One row in first table → Many rows in second table
- Example: One Customer → Many Orders

**2. Many-to-One**
- Reverse of one-to-many
- Many rows → One row

**3. One-to-One**
- One row → One row
- Less common

**4. Many-to-Many**
- Many rows → Many rows
- Requires bridge table

### Creating Relationships

**Auto-Detect:**
- Power BI often detects relationships automatically
- Check in Model view

**Manual Creation:**
1. Go to Model view
2. Drag field from one table to related field in another
3. Set relationship properties:
   - Cardinality (One-to-Many, etc.)
   - Cross filter direction
   - Make this relationship active

### Relationship Properties

**Cardinality:**
- One-to-Many (1:*)
- Many-to-One (*:1)
- One-to-One (1:1)
- Many-to-Many (*:*)

**Cross Filter Direction:**
- Single: Filters flow one direction
- Both: Filters flow both directions

## 9. Basic Calculations

### Simple Calculations

**Percentage:**
```dax
Sales % = DIVIDE(SUM(Sales[Amount]), SUM(AllSales[Amount]))
```

**Growth:**
```dax
Growth = [Current Period] - [Previous Period]
Growth % = DIVIDE([Growth], [Previous Period])
```

**Running Total:**
```dax
Running Total = 
CALCULATE(
    SUM(Sales[Amount]),
    FILTER(
        ALL(Sales[Date]),
        Sales[Date] <= MAX(Sales[Date])
    )
)
```

### Date Calculations

**Year-to-Date:**
```dax
YTD Sales = TOTALYTD(SUM(Sales[Amount]), Sales[Date])
```

**Previous Year:**
```dax
PY Sales = CALCULATE(SUM(Sales[Amount]), SAMEPERIODLASTYEAR(Sales[Date]))
```

## 10. Best Practices

### Data Model Best Practices

1. **Star Schema Design**
   - Fact table in center
   - Dimension tables around it
   - Clean, organized structure

2. **Proper Relationships**
   - Create all necessary relationships
   - Use correct cardinality
   - Set appropriate cross-filter direction

3. **Naming Conventions**
   - Clear, descriptive names
   - Consistent naming
   - Avoid spaces (use underscores)

### Visualization Best Practices

1. **Choose Right Chart Type**
   - Match chart to data type
   - Consider your audience
   - Keep it simple

2. **Use Consistent Colors**
   - Same color for same category
   - Use color meaningfully
   - Consider colorblind users

3. **Clear Labels**
   - Descriptive titles
   - Label axes clearly
   - Include units

4. **Limit Data Points**
   - Don't overcrowd charts
   - Use filters to focus
   - Group when necessary

### Performance Tips

1. **Import Only Needed Data**
   - Remove unnecessary columns
   - Filter at source when possible
   - Use DirectQuery for large datasets

2. **Optimize Measures**
   - Avoid complex calculations in visuals
   - Use measures instead of calculated columns when possible
   - Test performance

3. **Limit Visuals per Page**
   - Too many visuals slow down report
   - Use bookmarks for navigation
   - Consider multiple pages
